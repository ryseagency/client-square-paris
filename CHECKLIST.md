# Checklist de livraison

Vérifications obligatoires avant de marquer une page/section comme livrée. Regroupée par les 4 critères absolus définis dans `CLAUDE.md`.

## 1. Pixel-perfect

- [ ] `scripts/compare-pages.mjs` lancé sur desktop ET mobile
- [ ] Breakpoints testés : 375 / 768 / 1024 / 1440 / 1920
- [ ] Aucune hot zone > 10% sur les pages portées
- [ ] Computed styles diff (`scripts/compare-styles.mjs`) propre sur sélecteurs critiques
- [ ] Animations / transitions / hover states identiques au Lovable
- [ ] Comportement scroll (sticky, parallax, reveal) identique
- [ ] États interactifs testés : hover, focus, active, disabled

## 2. SEO / SERP

- [ ] Lighthouse SEO = 100 sur home + product + collection
- [ ] Lighthouse Performance ≥ 80 mobile, ≥ 90 desktop
- [ ] Lighthouse Accessibility ≥ 95
- [ ] Title 50-60 chars selon patterns `@CLIENT.md`
- [ ] Description 140-160 chars selon patterns `@CLIENT.md`
- [ ] Canonical présent sur toutes les pages
- [ ] OpenGraph complet : type, title, description, url, site_name, image (1200px)
- [ ] Twitter Card `summary_large_image`
- [ ] Sitelinks Search Box JSON-LD (`WebSite` + `SearchAction`) sur homepage
- [ ] Hreflang slot prêt même mono-locale
- [ ] Robots non touché côté code (passer par Search & Discovery)
- [ ] Navigation : `<nav aria-label>` + `role="menubar"` + `role="menuitem"` (pas de `SiteNavigationElement` deprecated)

## 3. GEO (si lieu physique)

- [ ] JSON-LD `Organization` validé sur https://validator.schema.org/
- [ ] JSON-LD `LocalBusiness` (ou `Store`) validé sur le même validator
- [ ] PostalAddress complète : streetAddress, addressLocality, postalCode, addressRegion, addressCountry
- [ ] geo `latitude` + `longitude` présents et corrects (vérifier sur Google Maps)
- [ ] `openingHoursSpecification` détaillée par jour
- [ ] `aggregateRating` si avis (ratingValue + reviewCount)
- [ ] `telephone` au format international (+33...)
- [ ] NAP strictement identique partout : footer + page contact + JSON-LD + Google Business Profile
- [ ] `sameAs` avec URLs réseaux sociaux

## 4. AEO (Answer Engine Optimization)

- [ ] `FAQPage` JSON-LD UNIQUEMENT sur sections affichant les Q/R visibles (sinon pénalité Google)
- [ ] `Product` JSON-LD sur fiches produit : name, image[], description, sku, brand, offers, aggregateRating
- [ ] `BreadcrumbList` sur collection + product + page
- [ ] `Speakable` (`SpeakableSpecification`) sur FAQ + À propos avec sélecteur `[data-speakable]`
- [ ] H2 sémantiques répondant à des questions implicites
- [ ] Première phrase de chaque section autonome (les LLM citent les premières phrases)
- [ ] Données factuelles dans `<dl>` ou tableaux structurés
- [ ] Rich Results Test (https://search.google.com/test/rich-results) → 0 erreur sur home + product + FAQ + contact

## Theme Store-friendly (transverse)

- [ ] Theme Check 0 error / 0 warning : `cd shopify-theme && shopify theme check --severity error`
- [ ] Tous les `name` schema ≤ 25 chars
- [ ] Tous les `range` valides : `default` sur step exact, ≤ 101 steps
- [ ] Aucun `default: ""` sur text/textarea/richtext/inline_richtext
- [ ] Toutes images ont `alt` non vide (settings + fallback `product.title` / `image.alt`)
- [ ] Tous `<a>` ont texte ou `aria-label`
- [ ] Tous `<button>` ont `type` explicite
- [ ] Skip-link `<a href="#MainContent">` + `<main id="MainContent" tabindex="-1">`
- [ ] Cart fonctionnel sans JS (fallback `/cart`)
- [ ] Newsletter via `{% form 'customer' %}` natif (pas Klaviyo en dur côté code)
- [ ] Liens internes via `routes.*_url` (jamais `/cart` en dur)
- [ ] Templates JSON sans section orpheline (toutes référencées dans `sections`)
- [ ] `settings_schema.json` propre, sections groupées par `name`

## Code propre (vérification finale)

- [ ] Aucun commentaire IA / origine (`claude`, `lovable`, `generated by`, `// AI`)
- [ ] Aucun TODO / FIXME / XXX / HACK
- [ ] Aucun `console.log` / `debugger` / `print()` de debug
- [ ] Aucune variable qui leak l'origine (`aiGenerated`, `lovableTmp`, `_v2_final`)
- [ ] Aucun code commenté laissé "au cas où"
- [ ] Indentation cohérente (2 espaces partout)

## Question finale (Boris Cherny)

> *« Un staff engineer Shopify validerait-il ça en l'état ? »*

Si non → la livraison n'est pas terminée, retour aux items qui clochent.
